# 投资分析框架

> VC 和创业投资中常用的分析框架和评估方法。

---

## 一、市场分析

### TAM/SAM/SOM 框架

```
TAM (Total Addressable Market)
 └── SAM (Serviceable Addressable Market)
      └── SOM (Serviceable Obtainable Market)
```

| 层级 | 定义 | 计算方式 |
|------|------|----------|
| **TAM** | 总可触达市场 | 如果100%市场份额的收入 |
| **SAM** | 可服务市场 | 当前产品能覆盖的部分 |
| **SOM** | 可获得市场 | 短期内现实可获得的份额 |

**两种估算方法**:
- **自上而下**: 从行业报告出发，逐层缩小
- **自下而上**: 从单价×客户数出发，逐层放大

**注意事项**:
- TAM 应该足够大 (通常 > $1B)
- SAM/SOM 的假设要现实
- 市场是动态的，可以创造新市场

---

### 市场类型分析

| 市场类型 | 特点 | 策略 |
|----------|------|------|
| **现有市场** | 已有明确竞争者 | 差异化或更好的产品 |
| **重新细分市场** | 在现有市场切出细分 | 专注特定场景/客户 |
| **新市场** | 没有直接竞争者 | 教育市场，定义品类 |
| **克隆市场** | 复制其他地区成功模式 | 本地化适配 |

---

### 竞争格局分析

**波特五力模型**:
1. **现有竞争者** - 市场集中度，竞争强度
2. **潜在进入者** - 进入壁垒高低
3. **替代品威胁** - 是否有其他解决方案
4. **供应商议价能力** - 上游依赖程度
5. **买家议价能力** - 客户集中度

**竞争壁垒评估**:
| 壁垒类型 | 强度 | 例子 |
|----------|------|------|
| 网络效应 | ★★★★★ | 社交网络、平台 |
| 规模效应 | ★★★★☆ | 云计算、制造业 |
| 切换成本 | ★★★★☆ | 企业软件、SaaS |
| 数据优势 | ★★★★☆ | AI 公司 |
| 品牌认知 | ★★★☆☆ | 消费品牌 |
| 技术专利 | ★★★☆☆ | 生物医药 |
| 监管壁垒 | ★★★☆☆ | 金融、医疗 |

## 二、商业模式分析

### SaaS 商业模式画布

```
┌─────────────────────────────────────────────────────────────┐
│ 价值主张: 解决什么问题？为什么选择你？                          │
├─────────────────────────────────────────────────────────────┤
│ 收入模式       │ 定价策略        │ 单位经济                    │
│ - 订阅        │ - 按席位        │ - LTV/CAC > 3              │
│ - 按量        │ - 按用量        │ - CAC Payback < 12月       │
│ - 混合        │ - 阶梯定价      │ - Gross Margin > 70%       │
├─────────────────────────────────────────────────────────────┤
│ 获客渠道                      │ 留存策略                      │
│ - PLG (产品驱动)              │ - NRR > 120%                 │
│ - SLG (销售驱动)              │ - Churn < 5% 年              │
│ - 内容营销                    │ - 扩展收入                    │
└─────────────────────────────────────────────────────────────┘
```

### 增长策略对比

| 策略 | PLG | SLG |
|------|-----|-----|
| **定义** | 产品驱动增长 | 销售驱动增长 |
| **获客** | 自助注册、免费试用 | 销售团队主动触达 |
| **客户** | SMB、开发者 | 企业级 |
| **CAC** | 较低 | 较高 |
| **扩展** | 产品内扩展 | 客户成功团队 |
| **代表** | Slack, Notion, Figma | Salesforce, Workday |

### 平台/Marketplace 分析

**关键指标**:
| 指标 | 定义 | 健康值 |
|------|------|--------|
| GMV | 商品交易总额 | 持续增长 |
| Take Rate | 抽成比例 | 10-30% |
| Liquidity | 流动性 | 供需匹配度高 |
| LTV/CAC | 按供需分别计算 | > 3 |

**飞轮效应**:
```
更多卖家 → 更多商品 → 更多买家 → 更多交易 → 更多卖家...
```

## 三、财务指标分析

### SaaS 核心指标

**收入指标**:
| 指标 | 公式 | 健康值 |
|------|------|--------|
| ARR | MRR × 12 | 持续增长 |
| Net New ARR | 新增 + 扩展 - 流失 | 正向增长 |
| NRR | (期初ARR + 扩展 - 流失) / 期初ARR | > 120% |
| GRR | (期初ARR - 流失) / 期初ARR | > 90% |

**效率指标**:
| 指标 | 公式 | 健康值 |
|------|------|--------|
| LTV | ARPU × 毛利率 × 客户生命周期 | - |
| CAC | 销售营销费用 / 新客户数 | - |
| LTV/CAC | LTV / CAC | > 3 |
| CAC Payback | CAC / (ARPU × 毛利率) | < 12 月 |
| Magic Number | Net New ARR / 上季度销售营销费用 | > 0.75 |
| Burn Multiple | 净烧钱 / Net New ARR | < 2 |

### Rule of 40

```
Rule of 40 = 收入增长率 (%) + 利润率 (%)

> 40%: 优秀
30-40%: 良好
< 30%: 需改进
```

**不同阶段的权衡**:
| 阶段 | 增长率 | 利润率 | Rule of 40 |
|------|--------|--------|------------|
| 早期 | 100% | -60% | 40% ✓ |
| 成长期 | 50% | -10% | 40% ✓ |
| 成熟期 | 20% | 20% | 40% ✓ |

### 估值倍数参考

**ARR 倍数** (2024 市场):
| 增长率 | NRR | 倍数 |
|--------|-----|------|
| > 100% | > 130% | 15-30x |
| 50-100% | 120-130% | 10-15x |
| 30-50% | 110-120% | 5-10x |
| < 30% | < 110% | 3-5x |

**影响因素**:
- 增长率（最重要）
- NRR / 留存
- 毛利率
- 市场规模
- 竞争格局
- 管理团队

## 四、团队评估

### 创始人评估维度

| 维度 | 问题 | 权重 |
|------|------|------|
| **领域专长** | 对行业有多深的理解？ | ★★★★★ |
| **过往经历** | 有成功创业或相关经历吗？ | ★★★★☆ |
| **执行能力** | 能把想法变成现实吗？ | ★★★★★ |
| **招人能力** | 能吸引优秀人才吗？ | ★★★★☆ |
| **学习能力** | 能快速学习和适应吗？ | ★★★★☆ |
| **抗压能力** | 能在逆境中坚持吗？ | ★★★★☆ |
| **诚信正直** | 值得信任吗？ | ★★★★★ |

### 团队完整性

**理想的创始团队**:
- **CEO**: 愿景、融资、对外
- **CTO**: 产品技术
- **COO/CFO**: 运营财务

**红旗信号**:
- 联合创始人股权严重不均
- 核心岗位长期空缺
- 早期员工大量离职
- 创始人之间有矛盾

### 尽调清单

**商业尽调**:
- [ ] 产品 Demo 和用户体验
- [ ] 客户访谈 (5-10 个)

## 六、投资决策框架

### 投资备忘录结构

```
1. 执行摘要
   - 公司简介
   - 投资金额和估值
   - 投资理由 (3-5 点)

2. 市场机会
   - TAM/SAM/SOM
   - 市场趋势
   - 竞争格局

3. 公司概况
   - 产品/服务
   - 商业模式
   - 核心指标

4. 团队
   - 创始人背景
   - 核心团队
   - 组织能力

5. 财务分析
   - 历史业绩
   - 财务预测
   - 资金使用

6. 风险与缓解
   - 主要风险
   - 缓解措施

7. 交易条款
   - 估值
   - 投资金额
   - 关键条款

8. 退出路径
   - 潜在买家
   - IPO 可能性
   - 预期回报
```

### 风险评估矩阵

| 风险类型 | 低 | 中 | 高 |
|----------|----|----|-----|
| **市场风险** | TAM 明确 | 市场在形成 | 完全新市场 |
| **竞争风险** | 清晰壁垒 | 有竞争但差异化 | 红海竞争 |
| **技术风险** | 已验证技术 | 需要迭代 | 技术突破 |
| **执行风险** | 成熟团队 | 有经验但新领域 | 首次创业 |
| **资金风险** | 盈利或低烧钱 | 需要融资 | 高烧钱高风险 |

### 退出分析

**退出路径**:
| 类型 | 时间 | 回报 |
|------|------|------|
| IPO | 7-10 年 | 最高，但不确定 |
| 并购 | 3-7 年 | 确定性较高 |
| Secondary | 随时 | 部分变现 |

**回报计算**:
```
回报倍数 = 退出价值 × 持股比例 / 投资金额

IRR = 年化回报率，考虑时间因素
```

## 七、投资人视角

### 好项目特征

1. **巨大市场**: TAM > $10B
2. **强劲增长**: 年增长 > 2-3x (早期)
3. **产品领先**: NPS 高，用户留存好
4. **优秀团队**: 有相关经验和执行力
5. **清晰壁垒**: 网络效应、数据、技术
6. **合理估值**: 与阶段和指标匹配
7. **明确退出**: 有清晰的上市或并购路径

### 红旗信号

- 创始人不诚实或夸大数据
- 核心团队频繁离职
- 客户集中度过高 (单一客户 > 20%)
- 严重依赖某个渠道
- 毛利率过低 (< 50%)
- 烧钱效率极差
- 股权结构混乱
- 法律或合规问题
